# 🤝 贡献指南

感谢您对15本考研书系列项目的关注！我们欢迎各种形式的贡献。

---

## 📋 贡献方式

### 1. 报告问题（Issues）

如果您发现了问题，请提交Issue：

**问题类型**：
- 🐛 Bug报告：代码错误、运行异常
- 📝 文档问题：错别字、表述不清、格式错误
- 💡 功能建议：新功能需求、改进建议
- ❓ 使用问题：不知道如何使用某个功能

**Issue模板**：
```markdown
### 问题描述
清晰描述问题是什么

### 复现步骤
1. 第一步...
2. 第二步...
3. 出现问题

### 预期行为
应该发生什么

### 实际行为
实际发生了什么

### 环境信息
- 操作系统：
- Python版本：
- 相关库版本：

### 截图（可选）
如果适用，添加截图帮助说明

### 补充信息
其他相关信息
```

---

### 2. 修复错误（Bug Fix）

**流程**：
1. Fork本仓库
2. 创建新分支：`git checkout -b fix/bug-description`
3. 修复错误
4. 添加测试（如适用）
5. 提交代码：`git commit -m "Fix: 修复了XXX问题"`
6. 推送分支：`git push origin fix/bug-description`
7. 创建Pull Request

**要求**：
- ✅ 确保代码风格一致
- ✅ 添加详细的提交说明
- ✅ 如果可能，添加测试用例
- ✅ 更新相关文档

---

### 3. 添加新功能（New Feature）

**流程**：
1. 先提交Issue讨论功能需求
2. 获得认可后，Fork仓库
3. 创建功能分支：`git checkout -b feature/feature-name`
4. 实现新功能
5. 编写文档
6. 添加示例代码
7. 提交PR

**要求**：
- ✅ 功能完整、可用
- ✅ 代码注释清晰
- ✅ 提供使用文档
- ✅ 包含示例代码
- ✅ 不破坏现有功能

---

### 4. 改进文档（Documentation）

文档改进包括：
- 修正错别字、语法错误
- 完善说明、增加示例
- 翻译文档
- 添加图表、示意图

**流程**：
1. Fork仓库
2. 创建文档分支：`git checkout -b docs/improvement`
3. 修改文档
4. 提交PR

**要求**：
- ✅ 语言清晰、准确
- ✅ 格式统一、美观
- ✅ 示例充分、易懂

---

### 5. 分享使用经验

您可以通过以下方式分享：
- 📝 撰写使用心得、学习笔记
- 🎥 录制视频教程
- 📊 展示学习成果、项目案例
- 💬 在社区解答其他用户的问题

## 📐 代码规范

### Python代码

**风格指南**：
- 遵循PEP 8规范
- 使用4个空格缩进（不用Tab）
- 最大行长度：100字符
- 使用有意义的变量名

**示例**：
```python
def calculate_normal_depth(Q, b, m, n, S0):
    """
    计算明渠均匀流正常水深
    
    Parameters:
    -----------
    Q : float
        流量 (m³/s)
    b : float
        底宽 (m)
    m : float
        边坡系数
    n : float
        曼宁系数
    S0 : float
        底坡
    
    Returns:
    --------
    float
        正常水深 (m)
    """
    # 实现代码...
    pass
```

**文档字符串**：
- 每个函数都要有docstring
- 说明参数、返回值、功能
- 提供使用示例

---

**格式要求**：
- 使用标准Markdown语法
- 代码块指定语言
- 表格格式规范
- 适当使用Emoji（不过度）

**示例**：
```markdown

**加粗文本**

```

## 🔍 Pull Request要求

### PR标题格式

```
<类型>: <简短描述>

类型：
- Fix: 错误修复
- Feature: 新功能
- Docs: 文档改进
- Style: 代码格式（不影响功能）
- Refactor: 重构
- Test: 测试相关
- Chore: 构建/工具相关
```

**示例**：
- `Fix: 修复学习助手CLI连续打卡统计错误`
- `Feature: 添加学习数据导出功能`
- `Docs: 完善FAQ中的Python环境配置说明`

---

### PR描述模板

```markdown
## 变更类型
- [ ] Bug修复
- [ ] 新功能
- [ ] 文档改进
- [ ] 代码优化
- [ ] 其他

## 变更说明
简要描述本次变更的内容和原因

## 相关Issue
关联的Issue编号：#XXX

## 测试情况
- [ ] 已在本地测试
- [ ] 测试覆盖率：XX%
- [ ] 测试环境：操作系统、Python版本

## 检查清单
- [ ] 代码遵循项目规范
- [ ] 已添加/更新文档
- [ ] 已添加测试用例
- [ ] 所有测试通过
- [ ] 不会破坏现有功能

## 截图（可选）
如果适用，添加截图展示变更效果

## 补充说明
其他需要说明的内容
```
